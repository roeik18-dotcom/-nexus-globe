AI Agent Architecture

Layers, concepts, execution model, and dependency map for modern AI Agent systems.

v2.0 · 2026
LAYER 01 Foundation What every agent is built on
Model (LLM)
LLM
PurposeCompute language — generate text, tool calls, and reasoning traces from a context window.
InputContext window (prompt · history · retrieved docs · tool results)
OutputText tokens, tool call instructions, chain-of-thought
RuntimeEvery inference step
מחשב שפה — מייצר טקסט, קריאות כלים ועקבות חשיבה מתוך חלון ההקשר.
Prompt
PROMPT
PurposeDefine task, persona, constraints, output format, and tools available to the model.
InputHuman-authored instructions · dynamic variables · tool schemas
OutputSystem prompt injected at the top of the context window
RuntimePer session or per request
מגדיר משימה, אישיות, מגבלות, פורמט פלט וכלים זמינים למודל.
Context
CONTEXT
PurposeSupply all working information to the model for a single inference step.
InputPrompt · conversation history · retrieved docs · tool results
OutputFull context window passed to the LLM
RuntimeAssembled fresh per inference step · bounded by model limit
מספק את כל המידע הפעיל למודל לשלב הסקה בודד. מוגבל בחלון ההקשר.
LAYER 02 Cognitive How the agent thinks
Reasoning
REASONING
PurposeWork through multi-step problems, weigh options, produce intermediate conclusions.
InputContext · goal · prior observations
OutputChain-of-thought, decision path, ranked options
Used byAgent · Planning
מעבד בעיות רב-שלביות, שוקל אפשרויות, ומייצר מסקנות ביניים מפורשות.
Planning
PLANNER
PurposeDecompose a complex goal into an ordered, executable sequence of subtasks.
InputGoal · Reasoning output · available tools
OutputOrdered step plan with dependencies
Used byAgent · Orchestrator
מפרק מטרה מורכבת לרצף מסודר של משימות משנה הניתנות לביצוע.
Memory
MEMORY
PurposePersist and retrieve information across interactions and agent iterations.
InputConversation history · tool results · episodic data
OutputRetrieved context injected into prompt or used by Planner
RuntimeRead before each step · written after each step
מאחסן ומאחזר מידע לאורך אינטראקציות ואיטרציות סוכן.
LAYER 03 Runtime What executes at run time
Agent
AGENT
PurposeAutonomous loop: perceive, plan, act, observe, repeat until goal met.
InputGoal · Tools · Memory · Model
OutputActions, tool calls, final response
RuntimeIterative loop until done or limit
לולאה אוטונומית: תפיסה, תכנון, פעולה, תצפית — עד להשגת המטרה.
Tools
TOOL
PurposeExecute a single callable action in the environment.
InputTool name · parameters from model output
OutputResult returned to model as an observation
RuntimeOn demand, per tool call
מבצע פעולה קריאה בודדת בסביבה ומחזיר תוצאה כתצפית.
Workflow
WORKFLOW
PurposeDeterministic, code-controlled sequence of LLM calls and tool steps.
InputPipeline spec · initial data
OutputStructured result from final step
RuntimeCode-driven — loop is NOT model-controlled
רצף דטרמיניסטי ונשלט-קוד של קריאות LLM וכלים — הלולאה בשליטת הקוד, לא המודל.
Observation
OBS
PurposeFeed tool results and environment state back to the model for next step.
InputTool output · environment state
OutputNew context added for next reasoning step
RuntimeAfter every tool execution
מזין תוצאות כלים ומצב סביבה חזרה למודל כקלט לשלב הבא.
LAYER 04 Orchestration & Infrastructure Coordination and external resources
Orchestrator
ORCH
PurposeCoordinate multiple agents, sub-agents, or pipeline steps.
InputHigh-level goal · agent registry · constraints
OutputRouted tasks, aggregated results, final synthesis
Used byMulti-Agent systems · complex pipelines
מתאם בין מספר סוכנים או שלבי צינור, מנתב משימות ומאגד תוצאות.
Multi-Agent
MULTI
PurposeDistribute work across specialized agents running concurrently or sequentially.
InputDecomposed subtasks · shared context
OutputParallel results aggregated by orchestrator
Used byComplex workflows · long-horizon tasks
מפיץ עבודה בין סוכנים מתמחים הפועלים במקביל או ברצף.
RAG
RAG
PurposeGround responses in retrieved external knowledge.
InputQuery · vector store · document corpus
OutputRelevant chunks injected into context before generation
RuntimeOn query, before each LLM call
מעגן תגובות בידע חיצוני שנשלף ממאגר וקטורים לפני היצירה.
MCP
MCP
PurposeStandardize how agents connect to external tools, data, and services.
InputTool definitions · server connections
OutputCallable tools exposed to the model via standard protocol
RuntimeSession init + per tool call
מתקנן חיבור סוכנים לכלים, נתונים ושירותים חיצוניים דרך פרוטוקול אחיד.
Execution Flow
①
Receive Goal
User input arrives; intent extracted
②
Build Context
Prompt assembled; memory + RAG injected
③
Reason
LLM produces chain-of-thought
④
Plan
Steps decomposed; dependencies mapped
⑤
Select Tool
Model emits tool call with parameters
⑥
Execute
Tool runs; result returned to model
⑦
Observe Result
Output evaluated; loop back to ③ if needed
⑧
Update Memory
Interaction stored for future retrieval
⑨
Return Response
Final output delivered to user
Knowledge Graph
Common Confusions
ConfusionCorrect Interpretation
Agent vs LLMThe LLM generates text. The Agent orchestrates — it decides when to call the LLM, which tools to invoke, and when the goal is met.
Memory vs ContextContext exists only for the current inference step. Memory persists across steps and sessions; it feeds into Context when retrieved.
Planning vs ReasoningReasoning thinks through a problem and weighs options. Planning sequences the output of reasoning into an executable order.
Tool vs WorkflowA Tool executes a single capability (one function call). A Workflow coordinates many calls — LLM + tool steps — in a defined sequence.
RAG vs MemoryRAG retrieves external, document-level knowledge from a vector store. Memory stores prior interaction history and episodic data.
MCP vs APIAn API exposes a service. MCP is a standard protocol that makes that service natively callable by AI models in a consistent format.
